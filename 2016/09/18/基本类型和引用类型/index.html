基本类型和引用类型

发表于 2016-09-18   |   分类于 javascript笔记   |  

数据类型

ES中的数据类型可以分为两种：

  • 简单基本数据类型（
    • Number
    • Bolean
    • String
    • Undefined
    • Null
  • 引用类型(对象)
    • Object
    • Array
    • function
    • 其他

基本数据类型的值是指简单的数据段，五种基本数据类型那是按值访问的，所以可以操作保存在变量中的值。
引用数据类型的值是保存在内存中的对象，在操作对象的时候，实际上只是在操作对象的引用，而不是操作内存中的对象。因为js不允许直接访问内存中的位置。

复制变量

变量复制能体现两种数据类型的值的本质区别。

上面代码中的上半段 ，a和b都是基本数据类型值； 把a赋值给b，并改变b的值，此时a依然保持不变。也就是说a与b是完全独立的。
但代码的下半段就不同了，person1和person1都是对象，把person1复制给person1，改变person1.name属性，发现person1.name属性也发生改变。person1 与 person2 完全相等，两个引用类型的值全等的条件是：他们指向同一个对象。也就是说person1和person1是同一对象的不同引用。

此时person1的内容发生改变 ，但是person1 并没有改变，而且两者并不全等。也就是person1在重新定义的时候指向了另外一个新的对象。

传递参数

ES中所有函数的参数传递都是按值传递的。

把实参传递给形参的时候，就和把值从一个变量复制到另一个变量一样。基本类型的值的传递如同基本类型变量复制一样，引用类型的传递，则如同引用类型复制一样。

传递基本类型的值的时候，被传递的值会被复制给一个局部变量（形参，也就是arguments对象中的一个元素），引用类型传递时，会把这个值在内存中的地址传递给形参，也就是说这个形参和实参指向同一个对象，形参的改变会反映到实参上（重新定义除外）。

由于参数传递是按照值进行传递的，并且形参是函数的局部变量，所以形参的改变并没有影响实参。

引用类型参数传递时，形参和实参指向同一个对象，所以形参的属性发生改变的时候会影响到实参。但这并不能说引用类型参数传递是按照引用传递的。

foo函数的形参确实发生改变，但是并没有反映到实参上，如果参数是按引用传递的，那么形参指向新的对象之后，实参也应该指向相应的对象，但实参并没有改变，所以是按照值传递的。

检测类型

之前已经接触过typeof操作符可以检测基本数据类型。但是检测引用数据类型时，只能得到object，并不能确定是哪一种具体的类型。因此ES提供了instanceof操作符，返回布尔值。
