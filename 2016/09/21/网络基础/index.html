TCP/IP协议

计算机网络设备之间的相互通信，肯定需要一种规则，因为这涉及到很多不同的硬件，不同的操作系统，这种规则就是协议(protocol)；
TCP/IP 协议就是与互联网相关联的协议的集合，也有人认为TCP/IP协议就是TCP/IP和IP这两种协议。

TCP/IP分层

TCP/IP协议按层可以分为四层：

  • 应用层
  • 传输层
  • 网络层
  • 数据链路层

应用层 （Application Layer）

应用层位于TCP/IP协议族的最上层，最靠近用户，决定了向用户提供服务时通信的活动。
应用层预存了各类通用的应用服务

  • FTP （File Transfer Protocol，文本传输协议）
  • DNS （Domain Name Sytem， 域名解析系统）
  • HTTP

传输层 （Transport Layer）

传输层在应用层上层，提供处于处于网络链接中的两台计算机的数据传输，有两个性质不同的协议 ： TCP （transmission Controll） 传输控制协议 和 UDP（User Data Protocol ）用户数据协议

网络层 （Internet Layer）

网络层用来处理在网络上流动的数据包。数据包时网络传输的最小数据单位。该层规定了通过怎样的路径到达对方的计算机，并把数据包传送给对方。
IP （Internet Protocol）网际协议

数据链路层

用来处理链接网络的硬件部分，包括操作系统，硬件的驱动，网卡，光纤等物理可见部分。

TCP/IP通信传输流程

利用TCP/IP协议族通信的时候，会通过分层顺序与对方进行通信，发送端从应用层往下走，接收端则向上走,
以HTTP举例：

  1. 发送端的客户端在应用层(HTTP协议)发出一个HTTP请求，也就是（HTTP报文）
  2. 在传输层（TCP/IP协议）把应用层处传来的数据进行分割，并在各个报文上面打上标记序号和端口号发送给网络层；
  3. 在网络层（IP协议），增加作为通信目的地的MAC地址之后转发给链路层，
  4. 接受的服务端在链路层接收到数据，按顺序往上层发送，直到应用层，才算真正就到发送端发来的HTTP请求;

发送端在层与层之间传输数据，每经过一层，都会为数据打上该层所属的首部信息。接收端在层与层之间传输数据时，每层都会将对应的首部信息去掉。这种方法成为封装。

IP，TCP和DNS

负责传输的IP协议

IP协议（Internet Protocol）位于网路层，是协议族中非常重要的一环，之前经常接触IP地址，那么，IP和IP地址什么区别？
IP只是一种协议的名称；a
IP协议的作用是把各种数据传送给对方，要保证传送到对方哪里，需要满足各种条件，两个重要的条件就是正确的IP地址和MAC（Media Accss Condrol Address）；
IP地址明确了节点被分配到的地址，MAC地址是网卡所属的固定地址。IP地址可以和MAC地址进行配对，IP可以而MAC地址基本不变。

MAC地址是网络中各种网络设备的地址，每个网络设备都有一个唯一的地址。
假设现有一台路由器，含有一个WAN端口及四个LAN端口。它的WAN端口会有一个如 61.61.61.61的IP地址，也会有一个如00:0A:02:0B:03:0C的MAC地址。而它的四个LAN端口，会各分配到一个例如192.168.1.0/24这样的IP地址，四个端口各会有一个不同的MAC地址。 – 维基百科MAC地址

使用ARP协议借用MAC地址进行通信
IP之间的通信依赖MAC地址，在网络上，通信双方通信时通常要经过多台计算机和网络射别的中转。而在中转过程中，会利用下一站中转设备的MAV地址来搜索下一个中转目标，这里采用（Address Resolution Protocol），是路由解析地址的协议，根据通信双方的IP地址就能反查出对应的MAC；

确保传输可靠性的TCP协议

为了方便的传输数据，将大块的数据切分成多个报文段为单位的数据包进行管理。

三次握手

为了确保信息能到达目标处，TCP协议采取了三次握手的策略。握手过程中使用了TCP的标志—-SYN和ACK

发送端首先发送带有SYN标志的数据包给对方，接收端收到后回传一个带有SYN/ACK标志的数据包表示传达确认信息。最后发送端在传回一个带ACK标志的数据包，表示传输结束。

如果传输中断，TCP协议就会在此以相同的顺序发送相同的数据包。

DNS

DNS负责域名解析，和HTTP协议一样位于应用层。
为什么使用域名解析？
访问计算机时，既可以使用域名或主机名也可以使用IP地址。域名符合人们的记忆习惯，而IP地址是计算机擅长处理的。DNS据提供了域名到IP地址的双向查找服务。

各种协议与HTTP协议的关系

HTTP协议

HTTP协议也是用于客户端和服务端之间的通信，请求资源的就是客户端，提供资源的就是服务端。
HTTP协议通过请求和响应的交换来通信。客户端发送请求，服务端接收到请求之后响应该请求并返回响应。

GET：获取资源

GET方法用来请求访问以备URL识别的资源，指定资源经服务器解析之后返回响应。

持久链接

早期的HTTP协议中，每进行一次HTTP通信就要断开一次TCP链接。当然早起传输的数据比较小，这样也没太大问题。但是现在网页中的图片等资源日益增多，每次请求都会造成无谓的TCP连接的建立和断开，增加了开销。为了解决上述问题，HTTP/1.1出现了持久连接(HTTP Keep-alive)，只要任意一端没有提出断开，则保持TCP连接。

使用Cookie进行状态管理

引入Cookie之前，我们需要知道HTTP协议是不保存状态的协议，也就是说，HTTP协议本身不会保存请求和响应之间的通信状态。但是现代网页中都有保持登录的功能，非常方便这是怎么回事呢？
这是使用了Cookie技术，在请求报文和响应报文中写入参数来管理登陆状态。
Cookie会根据服务端发送的响应报文内的Set-Cookie的首部字段通知客户端保存Cookie，下次客户端再往服务器中发送请求的时候，客户端会自动在请求报文中添加Cookie的值后发送出去。
服务端接受到客户端发送来的Cookie数据后，会检查从哪个客户端中发来的请求，去服务器中的记录进行对比，得到之前的登录状态。
