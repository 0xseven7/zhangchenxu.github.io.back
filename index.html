正则

ES通过RegExp类型来支持正则表达式

pattern模式部分可以接收任何了简单或复杂的正则表达式。
flages标明正则表达式的行为：

  • g：表示global模式；模式将被应用于所有字符串，不会在检索到第一个字符串的时候停下来；
  • i ：表示case-sinsenstive忽略大小写；
  • m：multiline多行模式，在前一行末尾后查找下一行；

一个正则表达式就是一个模式和三个标志的组合体。模式中所使用的元字符必须是转义字符
元字符：( [ ( * $ ^ ? \ + ) ] )
转意字符： “\”引导元字符

创建正则实例
创建正则实例和其他实例有两种方法：new操作符加RegExp构造函数和字面量

在使用new操作符创建RegExp实例的时候，由于RegExp构造函数参数为字符串，字面量创建时用转义字符的，这里需要用双重转义字符。
RegExp实例的方法
两种方法exec()和test()：

  • exec()方法，返回包含第一个匹配项的数组。没有匹配的话返回null。
    exec()包含两个属性：
    • index会返回第一个匹配项在字符串中的位置；
    • input会返回应用正则表达式的字符串；
    • test模式与参数匹配的时候会返回true，否则为false;

上述正则表达式中/d3表示3位数字，这种方法用来检测用户输入的格式是否有效。

#####全面学习正则
JS中的正则只是介绍了通过RegExp创建正则表达式的方法，但是正则的复杂的用法没有提及，比如怎么验证邮箱的格式？
这里就通过学习如何验证邮箱格式来学习正。
首先，邮箱的格式是什么？username@xxx.xxx；注册网易邮箱的时候的提示：

邮箱用户名名首字符为字母或者数字，中间可以是下划线_、数字和字母，以字母或数字结尾。

上面我们看到有些元字符在表达式中只能使用转义字符，因为这些字符在正则中有特殊的含义：
特殊字符：有特殊意义的符号

  • ()标记一个子表达式的开始和结束位置。
  • ^ 在字符串的首部匹配，在[]使用时，表示不匹配方括号中的表达式；
  • $在字符末尾匹配；
  • .匹配除换行符 \n之外的任何单字符;
  • |指明两项之间的一个选择;
  • ?匹配前面的子表达式零次或一次，(即使多次使用也只匹配最多一次)常和*+配合使用；
  • \转义符号；
    限定符
  • + 匹配前面子表达式的次数为1次或多次，相当于{1,}
  • * 匹配前面子表达式的次数为0次或多次，相当于{0,}
  • {}标记限定符表达式的开始和结束{a,b}a表示最小次数，b表示最大次数(a<=b);

首先邮箱名首字符为数字或者字母：
/^([a-zA-Z1-9])+/
这里使用了^表示这段模式部分是在目标字符串的首部匹配的，+表示表达式至少出现一次。
邮箱用户名中可以有_字母和数字，但不是必须的，所以限定符使用了*
([a-zA-Z0-9_])*;
用户名末尾必须为字母或数字，而且末尾位是在@之前的：
([a-zA-Z0-9])+@;
服务器域名由两部分组成，其中.之后有两个或者三个字母；
([a-zA-Z0-9])+\.([a-z]){2,3}$/
.这里使用了转义字符，而且使用了$表示从字符串末尾开始匹配；
这样一个基本的邮箱的正则表达式就完成了，这暂时没发现什么问题。

TCP/IP协议

计算机网络设备之间的相互通信，肯定需要一种规则，因为这涉及到很多不同的硬件，不同的操作系统，这种规则就是协议(protocol)；
TCP/IP 协议就是与互联网相关联的协议的集合，也有人认为TCP/IP协议就是TCP/IP和IP这两种协议。

TCP/IP分层

TCP/IP协议按层可以分为四层：

  • 应用层
  • 传输层
  • 网络层
  • 数据链路层
应用层 （Application Layer）

应用层位于TCP/IP协议族的最上层，最靠近用户，决定了向用户提供服务时通信的活动。
应用层预存了各类通用的应用服务

  • FTP （File Transfer Protocol，文本传输协议）
  • DNS （Domain Name Sytem， 域名解析系统）
  • HTTP
传输层 （Transport Layer）

传输层在应用层上层，提供处于处于网络链接中的两台计算机的数据传输，有两个性质不同的协议 ： TCP （transmission Controll） 传输控制协议 和 UDP（User Data Protocol ）用户数据协议

网络层 （Internet Layer）

网络层用来处理在网络上流动的数据包。数据包时网络传输的最小数据单位。该层规定了通过怎样的路径到达对方的计算机，并把数据包传送给对方。

之前很长一段时间不能理解闭包，经过这几天搜集资料，也对闭包有了一点较深的认识。那么闭包是什么呢？

闭包是什么？

上面代码中函数bar被当作值返回，并且通过调用foo()将bar赋给了baz，并调用baz()，实际上是通过不同的标识符引用调用了内部的函数bar()。
当foo()执行结束之后，根据垃圾回收的机制，foo()的整个内部作用域都会被销毁，但是由于闭包的发生而阻止了这件事情的发生，之后内部作用域一直存在，被bar()所使用。而这个函数在定义的词法作用域之外进行成功调用，闭包使得函数可以访问定义时的作用域。
当函数记住并访问所在词法作用域时，就产生了闭包，即使函数是在当前词法作用域之外执行的。函数对持有对该定义域的引用这就是闭包。

数据类型

ES中的数据类型可以分为两种：

  • 简单基本数据类型（
    • Number
    • Bolean
    • String
    • Undefined
    • Null
  • 引用类型(对象)
    • Object
    • Array
    • function
    • 其他

基本数据类型的值是指简单的数据段，五种基本数据类型那是按值访问的，所以可以操作保存在变量中的值。
引用数据类型的值是保存在内存中的对象，在操作对象的时候，实际上只是在操作对象的引用，而不是操作内存中的对象。因为js不允许直接访问内存中的位置。

简单的编译原理

学简单的学习一下编译原理：
在传统编译语言流程中，程序中的一段源代码在执行之前会经历三个步骤，统称为编译；
这个过程会将有组成的字符串分解为有意义的代码块，这些代码块被称为(token)。
如var a = 2;会被分解为”var “、”a”、”=”、”2”、“;”。

  1. 解析/语法分析
    这个过程会把词法单元流转换为一个由元素逐级嵌套所组成的代表语法结构的树，抽象语法树Abstract Syntax tree(AST);
  2. 代码生成
    将AST转化为一组机器指令。

    引擎：从头到尾负责整个JS程序的编译和执行的过程；
    编译器：负责语法分析及代码生成；
    作用域：可以看作是一套规则，用来管理引擎如何在当前作用域以及嵌套的子作用域中根据标识符名称进行变量查找。
