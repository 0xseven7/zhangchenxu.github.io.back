正则表达式总结

去年在高程中学习过正则表达式, 由于那时候刚接触, 再加上前后使用的场景比较少, 忘了很多. 下面就根据使用场景深化总结一下正则表达式.

用途

正则表达式通过正则表达式分支规定了某种特定的规则, 根据这种规则可以匹配出字符串中的对应的字符, 进而进行后续操作.

  • [lastIndex] 下一次exec匹配的开始索引, 初始值为0;
  • [source] 正则表达式的源文本

结构

从正则对象的创建中可以看出, 正则表达式分为两部分,正则表达式分支和标识符;

正则表达式分支

一个正则表达式分支包含一个或者多个正则表达式序列, 被管道符|分割.
他们之间是’或’的关系, 只要有一个符合条件, 这个选择就被匹配.他按照先后顺序匹配这些序列;

正则表达式序列

一个正则表达式序列包含一个到多个正则表达式因子, 每个因子是否跟随一个量词,量词决定了这个因子出现的次数.如果没有量词限制, 该因子只会匹配一次

量词

量词跟随到正则因子的后面, 决定了因子被匹配的次数, 那么量词有哪些呢?

  • ? 银子后附加, 表示此因子是可选的, 出现0次或者1次;

-{a} 此因子匹配a次

  • {a,b} 此因子匹配至少a次, 最多b次, b不设定表示贪婪匹配

  • * 此因子匹配不限次数, 相当于{0,}

  • + 此因子最少匹配一次, 相当于{1, }

    正则因子

    一个正则表达式因子可以使一个字符, 一个由圆括号包围的组, 一个字符类或者是一个转义序列

    元字符

    任何不包括以下字符正则比啊大事, 都可以简单的匹配其自身:$ ( ) * + . ? [ \ ^ { |
    正则表达式的巨大魅力和强大力量都是来源于上述12个字符, 但是不包括], -和}, 因为], -字符只有[后使用才有意义, }在{之后使用才有意义.
    转义
    既然这12个字符有特殊的含义, 那么如果想要匹配这些字符的话, 该怎么办的?
    答案就是通过在该字符前添加反斜线\来实现转义,}和]在对应的{}或[]中使用的时候也是需要转义的.
    另外, 对非数字和字母的字符加\进行转义并不影响正则的匹配, 但在不合时宜的时候进行转义回使正则表达式变得难以阅读.
    上述12个字符都有自己的特殊意义, 接着就来看看这些字符有什么神通使得正则表达式如此犀利?
    字符组
    使用方括号包裹的正则因子就是一个字符组, 如: [ae]匹配a或者e
    字符组用来匹配可能在字符列表中的单个字符, 在字符组中, 只有\ ^ - ]有特殊意义, 匹配的时候需要被转义, 而其他字符均可直接使用
  • -在字符组中表示一个范围,[a-zA-z]表示所有字母[0-9]所有个位数
  • ^左括号后跟着一个^, 表示排除正则字符组, 如 [^0-9]表示排除数字之外的其他字符
  • ] 避免出现歧义, 需转义
  • \ 转义字符
    简写(6个)
    有反斜线和一个字母组成的正则表达式会构成简写. 这种形式一共有6个:
  • \w 匹配单个的单词字符(word character), 也就是能疯狗出现在一个单词中的字符, 包括字母, 数字和下划线. 等价于[a-zA-Z0-9_]\
  • \W匹配除了[\w]之外的字符, 等价于[^\w]和[^a-zA-Z0-9_]
  • \d匹配数字, 相当于[0-9]
  • \D排除数字, 相当于[^0-9]
  • \s 匹配任意空白字符, 相当于[\f\n\r\t\u000B\u0020\u00A0\u2028\u2029],
  • \S 匹配排除[\s]之外的字符[^\f\n\r\t\u000B\u0020\u00A0\u2028\u2029]

注意这里\s和\S采用ASCII和Unicode两种编码方式, 所以会出现两种形式

  • \f换页符
  • \n换行符
  • \r回车符
  • \t制表符
匹配任意字符ss(. [\d\D] [\s\S] [\w\W])

.点号可以匹配除换行符(\n)之外的任意字符(对于Javascript);
那么如何包含所有字符呢?[\d]匹配所有数字, [\D]匹配所有非数字, 那么[\d\D]不就匹配全部字符了吗?
同理还有[\s\S]和[\w\W]

匹配首或尾(^ $)

^表示字符串的开始位置.
$表示字符串的结束位置
比如: >$, 标识字符串结尾是>

`/a/a/a/`.match(/^\/a/g); // [/a]
'/a/a/a/'.match(/\/a/g); // [/a, /a, /a]
'/a/a/a/'.match(/a\/$/g); // [a/]
'/a/a/a/'.match(/a\//g); // [a/, a/, a/]

正则表达式分组

捕获型分组(. . . )

正则表达式分组是一个被圆括号包围的分支.被匹配的分组都会被捕获, 第一个捕获性分组标号为1, 他匹配的文本副本会保存到result[1]中; 第二个捕获型分组编号2, 他匹配的文本副本会保存到result[2]… 依次类推

非捕获型分组(?. . .)

圆括号后面加一个?, 就表示此分组是非捕获型分组
